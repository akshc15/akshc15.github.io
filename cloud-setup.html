🔐 Connect to Your VM via SSH from M4 Mac

🧠
SSH = Secure Shell. It's like a magic tunnel from your Mac to your cloud VM. Once connected, every command you type runs on the cloud computer — not your Mac!
Terminal — M4 Mac
# Step 1: Move key file to .ssh folder and secure it
mv ~/Downloads/devops-key.pem ~/.ssh/
chmod 400 ~/.ssh/devops-key.pem
# chmod 400 = only YOU can read this file (required by SSH)

# Step 2: Connect! Replace IP with your EC2 Public IPv4
ssh -i ~/.ssh/devops-key.pem ubuntu@YOUR-EC2-IP

# First time: it asks "Are you sure you want to continue?"
# Type: yes then press Enter

# 🎉 You'll see this — you're now INSIDE your cloud VM!
ubuntu@ip-172-31-xx-xx:~$
💡
Make SSH easier — create a shortcut! Add this to ~/.ssh/config on your Mac:
~/.ssh/config
nano ~/.ssh/config

# Paste this (replace IP):
Host devops-vm
  HostName YOUR-EC2-IP
  User ubuntu
  IdentityFile ~/.ssh/devops-key.pem

# Save: Ctrl+X → Y → Enter

# Now connect with just this!
ssh devops-vm

💻 Connect VS Code on Your Mac to the Cloud VM

🤖
This is MAGIC! VS Code on your Mac will open files that are ACTUALLY stored on your cloud VM. You type code on Mac, it saves in cloud. Zero Mac storage used!
1
Install VS Code on your M4 Mac
Go to code.visualstudio.com → click Download for Mac (Apple Silicon) → open the downloaded .zip → drag VS Code to Applications folder
2
Install Remote-SSH Extension
Open VS Code → press Cmd+Shift+X (Extensions panel) → search Remote - SSH → click Install on the one by Microsoft (it has millions of installs)
3
Connect to your VM!
Press Cmd+Shift+P → type Remote-SSH: Connect to Host → click it → select devops-vm (from your SSH config) → a new VS Code window opens — this window is running ON your cloud VM! 🎉
4
Open a folder on the VM
In the new VS Code window → File → Open Folder → type /home/ubuntu → click OK. You're now browsing the cloud VM's files!
5
Open integrated terminal
Press Ctrl+` (backtick) → a terminal opens at the bottom → this terminal is running on your CLOUD VM — type pwd → it shows /home/ubuntu — you're in the cloud! ☁️
⚠️
VS Code tip: The bottom-left corner of VS Code shows a green SSH badge like SSH: devops-vm — this means you're connected to the cloud. If it shows nothing, you're working locally on your Mac.

⚙️ Install ALL DevOps Tools on Your Cloud VM

🤖
Run this ONE script and your VM gets: Docker, Git, kubectl, Terraform, Ansible, Helm, AWS CLI, Node.js, Python — everything for your entire DevOps journey!
Cloud VM Terminal (in VS Code)
# Run this on your Cloud VM terminal in VS Code
# ════ MASTER SETUP SCRIPT ════

# 1. Update system
sudo apt update -y && sudo apt upgrade -y

# 2. Install Docker
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu
# Log out and back in after this for Docker to work without sudo

# 3. Install Git
sudo apt install git -y
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 4. Install kubectl
curl -LO "https://dl.k8s.io/release/$(curl -sL https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

# 5. Install Terraform
sudo snap install terraform --classic

# 6. Install Ansible
sudo apt install ansible -y

# 7. Install Helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# 8. Install AWS CLI
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip && sudo ./aws/install

# 9. Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y

# ✅ Verify everything
docker --version && git --version && kubectl version --client && terraform --version
🎉
All tools are now on your cloud VM! Your Mac has zero extra software installed (except VS Code and SSH — tiny!). All the heavy stuff lives in the cloud.

SSH Keys — Connect Without Password
Set this up ONCE — never type passwords again
🧠
SSH key = two files. Private key (stays on your Mac, NEVER share!) + Public key (goes everywhere — GitHub, cloud VMs, servers). Like a lock and key — public key is the lock, private key opens it.
M4 Mac Terminal — One-time Setup
# Generate a strong SSH key pair (Ed25519 is best in 2026)
ssh-keygen -t ed25519 -C "your@email.com"
# Press Enter 3 times (accept defaults + no passphrase for convenience)

# See your public key (this goes to GitHub)
cat ~/.ssh/id_ed25519.pub
# Copy the output — starts with "ssh-ed25519 AAAA..."

# Add to GitHub:
# GitHub → Settings → SSH and GPG keys → New SSH key
# Title: "My M4 MacBook" → paste public key → Add SSH key

# Test GitHub connection
ssh -T git@github.com
# Should say: "Hi akshc15! You've successfully authenticated"

# Now clone with SSH (not HTTPS) — no password prompts!
git clone git@github.com:akshc15/my-repo.git

How senior engineers configure Git from day one
Run on Cloud VM Terminal
# Identity
git config --global user.name "Aksh C"
git config --global user.email "your@email.com"

# Default branch name (industry standard)
git config --global init.defaultBranch main

# Better diff tool
git config --global core.editor "code --wait"

# Auto-correct typos (git comit → git commit)
git config --global help.autocorrect 10

# Colorful output
git config --global color.ui auto

# Useful aliases
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.last "log -1 HEAD"
git config --global alias.lg "log --oneline --graph --all"

# See your full config
git config --list
