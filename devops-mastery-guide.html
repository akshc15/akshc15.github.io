CONCEPTS — What the Heck is All This?
Plain English explanations before you touch a single command
💡
Your mental model matters more than memorizing commands. Every tool here solves a specific pain. Learn the "WHY" first — the "HOW" becomes obvious.
🏗️ What is "Three-Tier" App?
Think of it as 3 separate layers: Frontend (React — what users see in browser), Backend (Node.js — the business logic, API), and Database (MongoDB — stores all data). They run independently and talk to each other over a network. This is how ALL real-world production apps are structured.
🐳 What is Docker?
Imagine packing your app + all its dependencies into a sealed box (container) that runs identically everywhere — your Mac, AWS, anywhere. No more "works on my machine" problems. A Dockerfile is the recipe. An image is the built box. A container is the running box.
⚙️ What is Kubernetes (K8s)?
You have 50 containers running. Who restarts them if they crash? Who scales them under load? Who routes traffic? Kubernetes is the orchestrator — it manages fleets of containers. AWS EKS is just Kubernetes hosted on AWS so you don't manage the control plane yourself.
🏗️ What is Terraform?
Instead of clicking around AWS console to create servers, you write code files (HCL language) describing what infrastructure you want. Terraform reads those files and creates/destroys/updates your AWS resources. This is Infrastructure as Code (IaC) — your infra is version-controlled, repeatable, and reviewable.
🔄 What is CI/CD (Jenkins)?
Every time a developer pushes code, you want it automatically tested, scanned for security issues, built into a Docker image, and deployed. Jenkins is the automation server that runs these "pipelines" for you. CI = Continuous Integration (build+test). CD = Continuous Deployment (release).
🌊 What is GitOps (ArgoCD)?
Instead of manually running kubectl apply to deploy, you commit Kubernetes manifests to Git. ArgoCD watches your Git repo and automatically syncs the cluster to match what's in Git. Git becomes the single source of truth for your cluster state.
🔐 What is DevSecOps?
Traditional DevOps = build fast, deploy fast. DevSecOps = build fast, deploy fast, securely. We embed security scanning at every step: SonarQube (code quality/vulnerabilities), OWASP Dependency Check (known vulnerable packages), Trivy (container image vulnerabilities). Security is not a gate at the end — it's baked in.
📊 Prometheus + Grafana?
Prometheus scrapes metrics from your app and cluster every few seconds (CPU, memory, HTTP request rates, error rates). Grafana is the beautiful dashboard that visualizes those metrics. Together they answer: "Is my app healthy right now?" Helm is the Kubernetes package manager — like apt/brew but for K8s apps.

PHASE 2 — Terraform Infrastructure Setup

🏗️ Phase 2 · Infrastructure as Code
🧠 Terraform Core Concepts
Provider = tells Terraform "talk to AWS". Resource = a thing you want (EC2, VPC, EKS). State file = Terraform's memory of what exists. terraform init = download providers. terraform plan = preview changes. terraform apply = actually create resources. terraform destroy = delete everything (SAVE MONEY!).

Step 2.1 — Deploy Jenkins Server via Terraform

1
Understand what the Terraform code creates
Open Jenkins-Server-TF/ in your editor. It creates: an EC2 instance (t2.large — Jenkins needs memory), a Security Group (firewall rules — opens port 22 for SSH, 8080 for Jenkins, 9000 for SonarQube), an IAM Role (so Jenkins can talk to AWS services). Create an S3 bucket for Terraform state (best practice)
S3 stores your terraform.tfstate so multiple people/machines can share it. Do this ONCE:

Create an EC2 Key Pair (to SSH into Jenkins)

PHASE 3 — Jenkins CI/CD Pipeline
Automate: test → scan → build → push → deploy on every git push
⚙️ Phase 3 · Continuous Integration
🧠 Jenkinsfile — The Pipeline as Code
A Jenkinsfile is a Groovy script that defines your pipeline stages. It lives in your Git repo. Jenkins reads it and executes each stage: Checkout Code → SonarQube Analysis → OWASP Check → Docker Build → Trivy Scan → Push to ECR → Update K8s manifest. This is Pipeline as Code — your CI/CD process is version-controlled.

Step 3.1 — Configure Jenkins Plugins & Tools

1
Install required Jenkins plugins
In Jenkins → Manage Jenkins → Plugins → Available plugins. Search and install each: Docker Pipeline, Amazon ECR, AWS Credentials, SonarQube Scanner, OWASP Dependency-Check, NodeJS, Pipeline: AWS Steps. Click Install without restart.
2
Configure AWS Credentials in Jenkins
Jenkins → Manage Jenkins → Credentials → Global → Add Credentials. Kind: AWS Credentials. ID: aws-creds. Paste your Access Key ID and Secret Access Key from IAM. This lets Jenkins push to ECR.
3
Configure SonarQube
SonarQube runs on same EC2 at port 9000. Open http://<EC2-IP>:9000. Default login: admin/admin. Change password. Then: Administration → Security → Users → Generate a token. Copy it. In Jenkins → Manage Jenkins → System → SonarQube servers → Add server URL http://<EC2-IP>:9000 and paste the token as a credential.
4
Add GitHub credential to Jenkins
Jenkins → Credentials → Add. Kind: Username with password. Username: your GitHub username. Password: a GitHub Personal Access Token (not your real password). Generate token at: GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → check repo scope.

Step 3.2 — The Jenkins Pipeline Stages (Explained)

Jenkinsfile — Frontend Pipeline (simplified walkthrough) pipeline {
  agent any
  stages {

    stage('Checkout') {
      // Pull your code from GitHub
    }

    stage('SonarQube Analysis') {
      // Scan source code → sends report to SonarQube dashboard
      // FAIL pipeline if quality gate fails (critical bugs found)
    }

    stage('OWASP Dependency Check') {
      // Scan package.json for known CVEs in npm packages
      // e.g. "you're using lodash 4.17.0 which has CVE-2021-xxxx"
    }

    stage('Docker Build') {
      // docker build -t frontend:${BUILD_NUMBER} .
      // Creates a Docker image from your Dockerfile
    }

    stage('Trivy Image Scan') {
      // trivy image --severity HIGH,CRITICAL frontend:tag
      // Scans the BUILT IMAGE for OS/library vulnerabilities
    }

    stage('Push to ECR') {
      // Tag image with ECR URI and push
      // aws ecr get-login-password | docker login
      // docker push 123456789.dkr.ecr.us-east-1.amazonaws.com/frontend:tag
    }

    stage('Update K8s Manifest') {
      // Update image tag in deployment.yaml in your Git repo
      // git commit && git push → ArgoCD detects this and redeploys
    }
  }
}

🌊 Phase 4 · GitOps / CD
🧠 How ArgoCD Works
ArgoCD runs inside your EKS cluster. It watches a Git repo (your Kubernetes-Manifests-file folder). When the Jenkins pipeline updates the image tag in deployment.yaml and pushes to Git, ArgoCD detects the change within seconds and automatically runs kubectl apply on your cluster. Zero manual deployment. Git commit = production deploy.

Kubernetes Questions

K8sWhat is the difference between a Pod, Deployment, and Service? +
A Pod is the smallest unit in K8s — one or more containers sharing network and storage. It's ephemeral (can die anytime). A Deployment manages Pods — it ensures the desired number are running, handles rolling updates, and recreates pods if they crash. A Service is a stable network endpoint that load balances traffic to pods (which have changing IPs). In my project, the frontend ran as a Deployment of 2 replicas behind a LoadBalancer Service to expose it to the internet.
K8sHow does Kubernetes handle a rolling update without downtime? +
When you update the image in a Deployment, K8s uses a RollingUpdate strategy: it starts a new pod with the new image, waits for it to become Ready (passes readiness probe), then terminates one old pod. It repeats this until all pods are updated. maxSurge: 1 and maxUnavailable: 0 means at any point, you have full capacity. In my project, ArgoCD triggered this automatically when Jenkins updated the image tag in the manifest file.
K8sWhat is a ConfigMap vs Secret? How did you use them? +
A ConfigMap stores non-sensitive config (env vars, config files) as plaintext. A Secret stores sensitive data (passwords, API keys) as base64-encoded values. In production you'd also encrypt secrets at rest. In my project, MongoDB connection strings were stored as K8s Secrets, injected into the Node.js backend pod as environment variables. This avoids hardcoding credentials in code.
K8sWhat is a PersistentVolume? Why does MongoDB need it? +
Container storage is ephemeral — when a pod dies, its data is gone. A PersistentVolume (PV) is a piece of storage (like an AWS EBS volume) provisioned independently. A PersistentVolumeClaim (PVC) is a request for storage by a pod. MongoDB absolutely needs a PVC because if the pod restarts without persistent storage, all your database data is lost. AWS EKS uses the EBS CSI driver to automatically provision EBS volumes when a PVC is created.

CI/CD & Jenkins Questions

CI/CDWhat's the difference between CI and CD? +
CI (Continuous Integration) is the practice of automatically building, testing, and scanning code on every commit. The goal is to catch bugs fast. CD (Continuous Delivery) means the software is always in a deployable state. Continuous Deployment takes it further — every passing build is automatically deployed to production with no human approval. In my project, Jenkins handled CI (build+scan) and ArgoCD handled CD (auto-sync to cluster).
CI/CDHow does your Jenkins pipeline ensure security? +
Three layers: (1) SonarQube SAST — static analysis of source code for vulnerabilities and code smells. Pipeline fails if quality gate doesn't pass. (2) OWASP Dependency Check — scans npm/package.json for known CVEs in third-party libraries. (3) Trivy image scan — scans the built Docker image for OS-level and package-level vulnerabilities. If CRITICAL vulnerabilities are found, the pipeline can be configured to fail, blocking deployment of vulnerable images.

Terraform / IaC Questions

IaCWhat is Terraform state? Why store it in S3? +
Terraform state (terraform.tfstate) is a JSON file that maps your Terraform config to real AWS resources. Without it, Terraform doesn't know what it already created. Storing state locally is fine for personal use but in teams, storing it in S3 with DynamoDB locking enables: (1) shared access by multiple engineers, (2) state versioning/history, (3) locking to prevent two people running terraform apply simultaneously which could corrupt state.
IaCWhat happens if you run terraform apply twice? +
Terraform is idempotent — it compares the desired state (your .tf files) with the current state (tfstate) and only makes changes if there's a diff. If nothing changed in your code, running apply again does nothing. This is a key benefit — you can safely re-run apply to ensure your infrastructure matches your code at any time.

DevSecOps / Monitoring Questions

GitOpsWhat is GitOps? Why is it better than imperative deployments? +
GitOps means Git is the single source of truth for both application code AND infrastructure state. ArgoCD watches the Git repo — if the cluster drifts from what Git says (e.g. someone manually ran kubectl), ArgoCD detects and corrects it. Benefits over imperative: (1) Full audit trail — every deployment is a Git commit with author/timestamp. (2) Easy rollback — just revert the commit. (3) No SSH/kubectl access needed for developers. (4) Separation of CI (Jenkins) and CD (ArgoCD).
MonitorWhat metrics would you alert on for a production app? +
In Prometheus/Grafana I'd alert on: (1) Pod crash-looping — container restart count > 5 in 10 min. (2) High CPU/Memory — node CPU > 80% for 5 min. (3) HTTP error rate — 5xx responses > 1% of total requests. (4) High latency — p99 response time > 2 seconds. (5) PVC usage — MongoDB disk > 80%. AlertManager (part of kube-prometheus-stack) can send these to Slack, PagerDuty, or email.
DockerWhat's the difference between COPY and ADD in Dockerfile? ENTRYPOINT vs CMD? +
COPY simply copies files from host to container — use this always unless you need ADD's extra features. ADD can additionally unpack tar archives and fetch URLs — but this makes Dockerfiles unpredictable. CMD provides the default command that can be overridden at runtime (docker run myimage bash). ENTRYPOINT sets the fixed executable — CMD then provides default arguments to it. Use ENTRYPOINT when the container IS a specific command (e.g. a Node.js server) and CMD for its default flags.
